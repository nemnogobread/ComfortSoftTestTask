# ComfortSoftTestTask

## Как запустить:
```
cd [path]
mvn clean package
java -jar target/ComfortSoftTestTask-0.0.1-SNAPSHOT.jar
```

## Как использовать:
Запрос отправлять на http://localhost:8080/get-n-statistics c параметрами 'path' и 'n'. Обратите внимение, что путь должен быть указан в URL кодировке, без знаков по типу ':/'. При использовании postman можно включить автокодирование, чтобы он делал это за вас.

Пример: `http://localhost:8080/get-n-statistics?path=L%3A%5Cdata.xlsx&n=10`

## Swagger: 
`http://localhost:8080/swagger-ui/index.html`

## Описание алгоритма: 
По сути дана задача на поиск N-ой порядковой статистики в неотсортированном массиве. Оптимальное решение для такой задачи - это использование вариации быстрой сортировки. Если точнее - QuickSelect. Этот алгоритм в среднем работает за O(n), но нужно уточнить, что в худшем случае (если pivot всё время выбирается неудачно, самый маленький или самый большой элемент в массиве) он может работать за O(n^2).

## Возможные улучшения
1. Сам алгоритм можно улучшить, оптимизируя выбор pivot элемента дла быстрого поиска (существует несколько известных вариантов оптимизации)
2. Использовать json сериализацию для ответа на запрос (например, ResponseEntity: `return ResponseEntity.status(HttpStatus.OK).body(responseBody);`, либо явно сериализовать ответ)
3. Сборку и развёртку приложения сделать через docker (но тогда нужно заранее знать путь до файла и монтировать его в исполняющую linux среду)
